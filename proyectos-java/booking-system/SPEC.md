# Especificacion de Booking System

## Requisitos minimos
- Varias entidades relacionadas.
- Reglas de negocio reales.
- Validaciones claras.
- Persistencia.
- Endpoints bien nombrados.
- README y pruebas.

## Ejemplo de reglas si eliges reservas
- No reservar un recurso dos veces en el mismo horario.
- No permitir fechas pasadas.
- Permitir consulta de disponibilidad.

## Ejemplo de entidades
- Resource
- Reservation
- User
- TimeSlot o fecha/hora de reserva
